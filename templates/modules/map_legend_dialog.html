  Map

Map types

pathfinder  supports 3 different map types. A map can consists of multiple systems (more info) and connections (more info). Maps are also referred to as "Chain Map" or "Chain".

  • Alliance map (can be seen by any alliance member)
  • Personal map (only you can use this kind of maps)
  • Public map (anyone can use this kind of maps)

Up to 5 different maps can be used simultaneously. Alliance maps only can be created with required rules.

Context Menu

right click somewhere on the map to open the context menu.

  • Show some basic map information
  • Open the "Add System" dialog and add a new system at the position where you right-clicked on the map
  • Delete all selected more info systems (if there are any selected systems) more info

Select methods

  • Drag-Select: Multiple systems can be selected by click + drag somewhere on the map and drag a frame around some systems
  • Single-Select: Individual systems can be selected by holding ctrl + click

Locked systems () canÂ´t be selected.

Grid snapping

For easier system arrangement, the "grid snapping" option can be activated on the "Map Menu" (). Dragged systems will snap then to a 20x20px grid.
A grid overlay will appear on the map.

Update counter

4

The map update counter starts counting backwards during map interaction (e.g. delete system, move system, ..). While the counter is active no data is pushed to the server. When the counter hits 0 all map date will be stored and any active pilot will receive the updates.
So there is no need for any save/edit buttons.

  System

Systems are represented by rectangle boxes on a map (more info). Pilots can interact with systems like "delete systems" (more info), "move systems around" (more info), ...

Activate

Any system in a map can be set as "active" by click somewhere at a system. Just one system per map can have an active state.
The active system has light yellow shaddow. Additional system information is shown below the map.

Name

Systems can be renamed with an alias by double-click the name. System aliases are stored server side and can be seen by any pilot with map access.

Security Status

Any "K-Space" system in "New Eden" has a fix "Security Status" between -1.0 and +1.0. "W-Space" systems always have a Security status of -1.0 but they can be distinguished into "Classes" [C1 - C6].
The color of the system label indicates the "Security Status" of a system

  • H high security system (1.0 - 0.5)
  • L low security system (0.4 - 0.1)
  • 0.0 Null security system (0.0 - -1.0)
  • C6, C5 heigh class wormhole (-1.0)
  • C4, C3 mid class wormhole (-1.0)
  • C6, C5 low class wormhole (-1.0)

System Status

The border color indicates the current status of a system.
The status can be changed by using the context menu (more info).

  • no status set (unknown)
  • friendly (in the possession of your corp/ally or allied entities)
  • occupied (hostiles structures found but no activity found)
  • hostile (hostile activity noticed recently)
  • empty (system is not occupied or active)
  • unscanned (system has unscanned signatures)

System Effect

Effects of wormhole systems are shown by a at the top right of a system

  • No effect
  • Magnetar
  • Red Giant
  • Pulsar
  • Wolfryet
  • Cataclysmic Variable
  • Blackhole

Active Pilots

If there are any active pilots in a system, there is a tooltip with the number of pilots.
Systems with active Pilots are bigger than empty systems. The name of a pilot is shown below the system name.
Hover the () icon in the systems head to get more detailed information (ship type, pilot name).

Move

Any system that is not "Locked" (more info) can be moved around by click + drag the systems security label (top right).
In order to move multiple systems at once, make sure all of them are selected (more info). Now you can move one systems and all others will follow.

Context Menu

right click a system on the map to open the context menu.

  • Open the "Add System" dialog and add a new system with a connection to this system
  • Lock this system more info
  • Set "Rally Point" for this system more info
  • Changes the status of this system more info
  • Delete this system and all connections more info

Locked system

Locked systems canÂ´t be selected, moved and deleted.

Rally Point

Systems can be marked as Rally Point.
Any active pilot will be informed about new "Rally Points" by a notice. Optional you can poke active pilots with "desktop push notifications" (more info).

Delete system

Any system that is not "Locked" (more info) can be deleted from a map.

  Connection (Stargate / Wormhole)

Connections between systems are represented by solid lines. Any connection requires two systems that are connected. A connection can be a "Stargate" or "Wormhole".

Connect Systems

Systems can be connected to each other by several ways

  • Drag-Connect: click a systems name and drag a new connection to any other system
  • New System-Connection: Add a system by using the context menu of a system (more info)

Connection Status

Wormholes will gain various statuses during its "Lifetime".
In addition to this, connections can get custom statuses.
The status can be changed by using the context menu (more info).

  • end of life (Wormhole is the end of its natural lifetime)
  • fresh (Has not yet had its stability significantly disrupted)
  • reduced (Wormhole had its stability reduced, but not to a critical degree) <50% mass left
  • critical (Wormhole is on the verge of collapse) <10% mass left
  • Frigate hole (Only the smallest ships pass through)

Connection Label

In addition to its "Lifetime Status" (more info), connections can be labeled with custom labels to supply further information.
Labels can be set by using the context menu (more info).

  • frig Frigate hole (Only the smallest ships pass through)
  • save mass save mass (Wormhole mass has to be saved)

Context Menu

right click a connection on the map to open the context menu.

  • Toggles this connection as "Frigate Hole" more info
  • Toggles this connection as "Preserve Mass" more info
  • Changes the status of this connection more info
  • Delete this connection more info

Frigate Hole

Wormholes can be labeled (more info) as "Frigate Hole". Only frigate-sized Spaceships can pass through a "Frigate Hole" (more info).

Preserve Mass

Wormholes can be labeled (more info) as "Preserve Mass". Let your mates know about critical connections that should be mass-saved (e.g. H security exits) (more info).

Delete connection

Existing connections can be detached by several ways

  • Drag-Connection: "Connection Endpoints" can be click + drag away from its current system
  • Delete-Connection: Delete a connection by using the context menu of a connection (more info)
  • Delete-System: Delete a system by using the context menu of a connection (more info)
  • Delete-Systems: Delete multiple systems by using the context menu of a map (more info)

Types

Success (success message after pilot interaction)
Information (non-important additional information)
Warning (important non-critical information)
Danger (very imprtant software error information)

Desktop push notifications

pathfinder supports desktop push notifications. The first time this kind of notification is send to client, a Browser security Question must be confirmed. In order to test desktop push notifications click the "Notification test" option in the main menu ().
Events can trigger this kind of notification

  • New Rally Point ( more info)
